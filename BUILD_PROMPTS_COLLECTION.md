### ВАРИАНТ 1: Простая сборка тестового APK

```
Привет! Мне нужно собрать тестовый APK для моего приложения Tazakkur.

Проект: React Native (Expo)
Локация: /путь/к/tazakkur/frontend

Выполни следующее:

1. Обнови версию приложения:
   - В app.json увеличь "version" с 1.0.0 на 1.0.1
   - В app.json увеличь "versionCode" (android) с 1 на 2

2. Проверь зависимости:
   - cd frontend
   - yarn install

3. Собери APK для тестирования:
   - eas build --platform android --profile preview

4. После сборки дай мне:
   - Прямую ссылку на скачивание APK
   - Размер файла
   - Номер сборки

Примечания:
- Bundle ID: com.tazakkur.app
- Иконка: mosque-logo.png
- Не забудь проверить что я залогинен в EAS (eas whoami)
```

### ВАРИАНТ 2: Сборка для публикации в Google Play

```
Мне нужно собрать production версию (AAB) для публикации в Google Play Store.

Проект: Tazakkur (React Native Expo)
Путь: /путь/к/tazakkur/frontend

Шаги:

1. Обнови версию:
   - app.json: version → 1.0.0
   - app.json: android.versionCode → 1

2. Проверь production настройки:
   - В .env должен быть production BACKEND_URL
   - Supabase ключи production

3. Собери AAB:
   - eas build --platform android --profile production

4. Проверь:
   - Размер AAB
   - Bundle ID: com.tazakkur.app
   - Подпись сертификата

5. Дай инструкции как загрузить в Google Play Console

Дополнительно:
- Если нужен keystore, создай новый
- Сохрани credentials безопасно
```

### ВАРИАНТ 4: Локальная сборка без облака

```
Помоги собрать APK локально на моём Mac, без использования EAS Build.

Проект: Tazakkur (Expo)
Путь: /путь/к/tazakkur/frontend

Шаги:

1. Проверь что установлено:
   - Android Studio
   - Java 17
   - Android SDK (API 34)

2. Настрой окружение:
   - Проверь ANDROID_HOME
   - Проверь PATH переменные

3. Prebuild:
   - npx expo prebuild --platform android

4. Собери APK:
   - cd android
   - ./gradlew assembleRelease

5. Найди APK:
   - android/app/build/outputs/apk/release/app-release.apk

6. Если ошибки - помоги исправить:
   - Gradle issues
   - SDK not found
   - Java version mismatch

Дополнительно:
- Нужна ли подпись APK?
- Как минимизировать размер?
```

### ВАРИАНТ 5: Обновление существующего APK (патч)

```
У меня есть собранный APK версии 1.0.0. Нужно собрать патч 1.0.1 с исправлениями.

Проект: Tazakkur
Изменения: Исправлены баги в экране Зикр и добавлена темная тема

Что сделать:

1. Проверь изменения в коде:
   - app/(tabs)/zikr.tsx
   - contexts/ThemeContext.tsx

2. Обнови версию:
   - 1.0.0 → 1.0.1
   - versionCode: 1 → 2

3. Собери новый APK:
   - eas build -p android --profile preview

4. Сравни с предыдущей версией:
   - Размер файла
   - Новые permissions (если есть)
   - Changelog

5. Дай рекомендации:
   - Нужно ли пользователям удалить старую версию?
   - Сохранятся ли данные?
```
